# ANDROID BACKEND WITH TELEMETRY & MAP 

Система для мониторинга параметров сотовой связи и геолокации в реальном времени. Приложение принимает данные от смартфона по протоколу ZMQ, визуализирует их на карте, строит графики сигналов и сохраняет историю в PostgreSQL.

## Основные возможности
* **Real-time сервер:** Прием и парсинг JSON-пакетов через ZeroMQ.
* **Отрисовка мапы:** Многопоточная подгрузка и кэширование тайлов OpenStreetMap на диске.
* **Аналитика:** Визуализация метрик RSRP, RSSI и SINR в реальном времени через ImPlot.
* **Хранение данных:** Логирование в JSON-файлы и трансляция в PostgreSQL.
* **Оптимизация:** Разделение логики на независимые потоки.

## Зависимости

Для корректной сборки необходимо установить следующие компоненты:

### Системные пакеты (macOS)
```bash
brew install sdl2 glew zeromq libpq curl cmake
```

### Сторонние библиотеки (в папке `third_party/`)
* **ImGui**: Интерфейс пользователя.
* **ImPlot**: Графики метрик.
* **nlohmann/json**: Парсинг входящих данных.
* **stb_image**: Загрузка PNG-тайлов карты.

## 🏗 Компиляция и запуск

Проект разработан и оптимизирован для работы под **macOS**. 
Сборка производится в автоматическом режиме с помощью скрипта setup.sh:

1.  Убедитесь, что у вас установлены все зависимости.
2.  Запустите скрипт сборки:
    ```bash
    chmod +x setup.sh
    ./setup.sh
    ```
3.  После завершения исполняемый файл будет находиться в папке сборки.

## Архитектура

Приложение спроектировано по модульному принципу, где каждый компонент отвечает за свой уровень логики:

| Модуль | Описание |
| :--- | :--- |
| **`main.cpp`** | Инициализация графического контекста (OpenGL 3.3 Core Profile), запуск потоков и главный цикл отрисовки. |
| **`telemetry.cpp / .h`** | Сетевой стек (ZMQ), логика парсинга и хранение истории сигналов в структуре `PciHistory`. |
| **`gui_app.cpp / .h`** | Высокоуровневая отрисовка интерфейса, окон телеметрии и интеграция ImPlot. |
| **`map_engine.cpp / .h`** | Движок карт: управление очередью загрузки тайлов через `libcurl` и работа с текстурами. |
| **`database.cpp / .h`** | Слой абстракции для работы с БД PostgreSQL через библиотеку `libpq`. |

### Модель потоков:
* **Main Thread**: Обработка событий SDL2 и рендеринг GUI.
* **Server Thread**: Фоновый прием данных по сети, обновление `LocationData` и запись в БД.
* **Tile Loader Thread**: Асинхронная загрузка изображений карт, исключающая фризы интерфейса при плохом интернет-соединении.

## Данные и логи
* **Кэш карт**: Тайлы сохраняются по пути `build/zoom/x/y.png`.
* **JSON Лог**: Все входящие пакеты записываются в файл `location_log.json` в корневой папке.
* **PostgreSQL**: Данные о каждой соте пишутся в таблицу для последующего анализа.
